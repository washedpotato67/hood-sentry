# Launch-day dashboards and alerts

Monitor availability, API latency and errors, PostgreSQL, Redis, queues, indexer lag, provider disagreement, reorgs, risk-scan failures, delivery failures, authentication anomalies, WAF blocks, `$SENTRY` transfers and supply, launchpad state, graduation, migration, liquidity, creator-fee claims, treasury transfers, impersonation reports, and entitlement failures.

Operational windows: pre-launch, launch moment, first 15 minutes, first hour, first 6 hours, and first day. Every critical monitor requires a named owner and actionable notification destination before enablement. Placeholders are not accepted as launch evidence.

Kill-switch owners must be assigned for mainnet writes, trading, token gating, launchpad integration, gas sponsorship, AI, webhooks, project claims, reports, and read-only mode. Until owners are assigned, Operations and Security share no implied authority and the related feature stays disabled.
