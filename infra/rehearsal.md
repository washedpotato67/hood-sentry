# Production-like rehearsal

Use a testnet or fixed-block fork only. Never use fake production data and never broadcast to mainnet.

Validate the environment, start services, index blocks, discover tokens and pools, process launchpad events, run risk scans, load a wallet portfolio, calculate P&L, create a watchlist and alert, link Telegram, test a webhook, claim a project, submit a report, test token entitlement with a fixture token, prepare and simulate a trade intent, then exercise provider failure, Redis failure, indexer restart, synthetic reorg, backup restore, and read-only mode.

Record evidence for every step, including service versions, source block, backup ID, duration, validation result, failure, and follow-up. Repeatability requires pinned configuration and an immutable rehearsal report.
