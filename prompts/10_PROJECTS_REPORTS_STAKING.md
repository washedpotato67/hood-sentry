# Prompt 10 — Projects, Reports, and Staking Integration

Connect the API, database, indexer, web, and contracts for:
- project wallet claim;
- project profile versions;
- contract-address verification;
- metadata content hash;
- project bonds;
- community reports;
- evidence upload;
- reporter bonds;
- moderation;
- appeals;
- access staking and premium tiers.

Requirements:
- onchain intent preparation and simulation;
- transaction tracking;
- no “safe” badge from identity verification;
- objective reason codes;
- no automatic AI slashing;
- admin decisions audited;
- report evidence privacy controls.

Acceptance:
- end-to-end testnet flows pass;
- duplicate claim and signature replay fail;
- transaction failure produces recoverable UI;
- role/permission tests pass;
- profile history is immutable and visible.
