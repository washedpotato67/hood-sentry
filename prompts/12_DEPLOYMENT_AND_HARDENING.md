# Prompt 12 — Deployment, Security, and Launch

Implement `docs/DEPLOYMENT_OPERATIONS.md`, `docs/SECURITY_THREAT_MODEL.md`, and the runbooks.

Create:
- Vercel/Railway configuration;
- Cloudflare security checklist;
- production environment schema;
- migrations and release scripts;
- observability dashboards/alerts;
- backups and restore test;
- provider failover;
- queue dead-letter tools;
- contract balance/admin monitors;
- incident feature flags;
- security headers;
- dependency/secret/container scans;
- SBOM;
- production smoke tests;
- launch status page.

Then execute the full release checklist and update `docs/IMPLEMENTATION_STATUS.md` with exact evidence: command output, deployment IDs, contract addresses, verification links, remaining disabled flags, and known risks.

Do not enable a write feature merely to satisfy the checklist. Enable it only when its gate passes.
